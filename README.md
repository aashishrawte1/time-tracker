# Time Tracker

A small personal time tracker: create projects, start/stop a timer while you work,
and see daily/weekly/monthly summaries of where your time went. Each account only
ever sees its own data.

## Stack

- **Frontend**: React + Vite + TypeScript + Tailwind CSS → deployed to **GitHub Pages**
- **Backend**: Node.js + Express + TypeScript + JWT auth → deployed to **Render**
- **Database**: MongoDB Atlas

Monorepo layout:

```
time-tracker/
  backend/   Express API
  frontend/  React app
```

## Local development

### Backend

```bash
cd backend
cp .env.example .env   # fill in MONGODB_URI and a JWT_SECRET
npm install
npm run dev             # http://localhost:4000
```

### Frontend

```bash
cd frontend
cp .env.example .env   # VITE_API_URL=http://localhost:4000/api
npm install
npm run dev             # http://localhost:5173
```

## Deployment

### 1. Backend → Render

GitHub Pages can only serve static files, so the API is deployed separately on Render
(free tier) even though the code lives in the same repo.

1. Go to [render.com](https://render.com) → New → Blueprint, and point it at this repo.
   Render will read `render.yaml` at the root and create the `time-tracker-api` web
   service automatically (root dir `backend/`).
2. In the service's **Environment** tab, set:
   - `MONGODB_URI` — your Atlas connection string (include a database name, e.g.
     `.../time-tracker?appName=...`)
   - `CORS_ORIGIN` — your GitHub Pages URL, e.g. `https://aashishrawte1.github.io`
   - `JWT_SECRET` is auto-generated by the blueprint; leave it as is.
3. Deploy. Note the resulting URL, e.g. `https://time-tracker-api.onrender.com`.

Free tier note: the service spins down after ~15 min idle, so the first request after
a period of inactivity takes 30-50s to wake up.

### 2. Frontend → GitHub Pages

1. In the repo **Settings → Pages**, set **Source** to **GitHub Actions**.
2. In **Settings → Secrets and variables → Actions → Variables**, add a repository
   variable `VITE_API_URL` = `https://time-tracker-api.onrender.com/api` (your Render
   URL + `/api`).
3. Push to `main` — `.github/workflows/deploy-frontend.yml` builds `frontend/` and
   publishes it to GitHub Pages automatically. The app will be live at
   `https://<your-username>.github.io/time-tracker/`.

Note the `base: "/time-tracker/"` in `frontend/vite.config.ts` and the matching
`basename` in `frontend/src/App.tsx` assume the repo is named `time-tracker`. If you
rename the repo, update both.

## API overview

All routes except `/api/auth/register` and `/api/auth/login` require
`Authorization: Bearer <token>` and only ever operate on the logged-in user's data.

- `POST /api/auth/register`, `POST /api/auth/login`, `GET /api/auth/me`
- `GET/POST/PUT/DELETE /api/projects`
- `GET /api/timer/active`, `POST /api/timer/start`, `POST /api/timer/stop`
- `GET/PUT/DELETE /api/entries`
- `GET /api/summary?range=daily|weekly|monthly&date=YYYY-MM-DD`
